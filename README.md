# Как добавить свои работы

## Шаг 1. Сделайте Fork репозитория

Нажмите кнопку **Fork** в правом верхнем углу страницы репозитория на GitHub.

После этого у вас появится собственная копия репозитория по адресу:
`https://github.com/ВАШ_USERNAME/название-репозитория`

---

## Шаг 2. Клонируйте fork к себе на компьютер

Скопируйте ссылку на **ваш** fork и выполните команду:

**SSH** *(если настроен SSH-ключ)*:
```bash
git clone git@github.com:ВАШ_USERNAME/название-репозитория.git
cd название-репозитория
```

> Замените `ВАШ_USERNAME` и `название-репозитория` на свои значения.

Затем добавьте оригинальный репозиторий как `upstream` — это позволит подтягивать обновления от других участников:

```bash
git remote add upstream git@github.com:GlebLarkin/Black-cocoa.git
```

Проверить, что всё настроено правильно:

```bash
git remote -v
# origin    https://github.com/ВАШ_USERNAME/название-репозитория.git (fetch/push)
# upstream  https://github.com/ORIGINAL_OWNER/название-репозитория.git (fetch/push)
```

---

## Шаг 3. Создайте новую ветку

> [!WARNING]
> **Не работайте в ветке `main`.** Все изменения вносятся только через отдельную ветку.

Рекомендуемый формат имени ветки: `lab-XX-username`  
*(например: `lab-03-ivanov`)*

```bash
git checkout -b lab-XX-username
```

## Шаг 5. Сделайте коммит и запушьте изменения

```bash
# Добавьте все изменения в индекс
git add .

# Создайте коммит с описанием
git commit -m "Add lab XX — ваше имя"

# Отправьте ветку в свой fork
git push
```

> [!CAUTION]
> **Не изменяйте и не удаляйте чужие работы.**  
> (вы все равно не сможете)

---

## Шаг 6. Создайте Pull Request

1. Перейдите на страницу **вашего** fork на GitHub.
2. Нажмите кнопку **Compare & pull request** (появится автоматически после пуша).
3. Убедитесь, что PR направлен в ветку `main` **оригинального** репозитория.
4. Напишите краткое описание и нажмите **Create pull request**.

После отправки ваш Pull Request будет проверен, и при отсутствии замечаний — влит в основную ветку.
